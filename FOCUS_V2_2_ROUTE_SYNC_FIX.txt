LifeOS Focus Studio v2.2 - Route Synchronization Fix

Problem fixed:
BuildError: Could not build url for endpoint 'focus_bp.save_preferences'

Why it happened:
The Focus Studio template and focus route file were not from the same version, or the preferences endpoint was not registered under the expected name.

Replace BOTH files:
1. routes/focus_routes.py
2. templates/focus_mode.html

This patch:
- Restores the complete Focus Studio v2.2 route file.
- Registers the preferences route explicitly as endpoint="save_preferences".
- Uses /focus/preferences directly in the form action to prevent template URL-build failures.
- Includes preferences, analytics, achievements, coach data, timers, cycles, distractions, reviews, and all v2.2 endpoints.

No SQL migration is needed.

After replacement, fully stop Python and restart:
  taskkill /F /IM python.exe
  python app.py

Optional verification in PowerShell:
  flask --app app routes | findstr focus

You should see a line for:
  /focus/preferences
