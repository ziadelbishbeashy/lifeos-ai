LifeOS Focus Studio v2.2 — Template Context Fix

Problem fixed:
- jinja2.exceptions.UndefinedError: 'preferences' is undefined
- jinja2.exceptions.UndefinedError: 'analytics' is undefined
- Prevents the same error for achievements, coach, and recent_sessions.

Replace these exact files in your LifeOS project:
1. routes/focus_routes.py
2. templates/focus_mode.html

IMPORTANT: Stop every running Python process before starting LifeOS again.
On Windows PowerShell:
    taskkill /F /IM python.exe
    python app.py

Why this happened:
The browser loaded the new v2.2 template, while Flask was still using an older
focus_routes.py module that did not provide every v2.2 template variable.

The corrected route passes:
- preferences
- analytics
- achievements
- coach
- recent_sessions

The corrected template also includes safe defaults so a missing optional context
value cannot crash the page during future upgrades.

No SQL migration is required for this fix.
